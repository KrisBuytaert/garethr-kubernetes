lib/puppet/type/kubernetes_object_reference.rb

Overview

ObjectReference contains enough information to let you inspect or modify the referred object.

Parameter Summary

  • apiVersion (Property) -

    API version of the referent.

  • fieldPath (Property) -

    If referring to a piece of an object instead of an entire object, this string should contain a valid JSON/Go field access statement, such as desiredState.manifest.containers. For example, if the object reference is to a container within a pod, this would take on a value like: 'spec.containersname' (where 'name' refers to the name of the container that triggered the event) or if no container name is specified 'spec.containers' (container with index 2 in this pod). This syntax is chosen only to have some well-defined way of referencing a part of an object.

  • kind (Property) -

    Kind of the referent. More info: releases.k8s.io/HEAD/docs/devel/api-conventions.md#types-kinds

  • name (Property) -

    Name of the referent. More info: releases.k8s.io/HEAD/docs/user-guide/identifiers.md#names

  • name (Parameter) -

    Name of the object_reference.

  • namespace (Property) -

    Namespace of the referent. More info: releases.k8s.io/HEAD/docs/user-guide/namespaces.md

  • resourceVersion (Property) -

    Specific resourceVersion to which this reference is made, if any. More info: releases.k8s.io/HEAD/docs/devel/api-conventions.md#concurrency-control-and-consistency

  • uid (Property) -

    UID of the referent. More info: releases.k8s.io/HEAD/docs/user-guide/identifiers.md#uids
