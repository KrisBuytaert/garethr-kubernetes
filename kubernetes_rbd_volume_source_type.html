Puppet Type: kubernetes_rbd_volume_source_type

lib/puppet/type/kubernetes_rbd_volume_source.rb

Overview

RBDVolumeSource represents a Rados Block Device Mount that lasts the lifetime of a pod

Parameter Summary

  • fsType (Property) -

    Filesystem type of the volume that you want to mount. Tip: Ensure that the filesystem type is supported by the host operating system. Examples: 'ext4', 'xfs', 'ntfs'. More info: releases.k8s.io/HEAD/docs/user-guide/volumes.md#rbd

  • image (Property) -

    The rados image name. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it

  • keyring (Property) -

    Keyring is the path to key ring for RBDUser. Default is /etc/ceph/keyring. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it

  • monitors (Property) -

    A collection of Ceph monitors. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it

  • name (Parameter) -

    Name of the rbd_volume_source.

  • pool (Property) -

    The rados pool name. Default is rbd. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it.

  • readOnly (Property) -

    ReadOnly here will force the ReadOnly setting in VolumeMounts. Defaults to false. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it

  • secretRef (Property) -

    SecretRef is name of the authentication secret for RBDUser. If provided overrides keyring. Default is empty. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it

  • user (Property) -

    The rados user name. Default is admin. More info: releases.k8s.io/HEAD/examples/rbd/README.md#how-to-use-it
