Overview

ServiceAccount binds together: * a name, understood by users, and perhaps by peripheral systems, for an identity * a principal that can be authenticated and authorized * a set of secrets

Parameter Summary

  • imagePullSecrets (Property) -

    ImagePullSecrets is a list of references to secrets in the same namespace to use for pulling any images in pods that reference this ServiceAccount. ImagePullSecrets are distinct from Secrets because Secrets can be mounted in the pod, but ImagePullSecrets are only accessed by the kubelet. More info: releases.k8s.io/HEAD/docs/user-guide/secrets.md#manually-specifying-an-imagepullsecret

  • metadata (Property) -

    Standard object's metadata. More info: releases.k8s.io/HEAD/docs/devel/api-conventions.md#metadata

  • name (Parameter) -

    Name of the service_account.

  • secrets (Property) -

    Secrets is the list of secrets allowed to be used by pods running using this ServiceAccount. More info: releases.k8s.io/HEAD/docs/user-guide/secrets.md
