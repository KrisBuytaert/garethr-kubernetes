lib/puppet/type/kubernetes_security_context.rb

Overview

SecurityContext holds security configuration that will be applied to a container.

Parameter Summary

  • capabilities (Property) -

    The linux kernel capabilites that should be added or removed. Default to Container.Capabilities if left unset. More info: releases.k8s.io/HEAD/docs/design/security_context.md#security-context

  • name (Parameter) -

    Name of the security_context.

  • privileged (Property) -

    Run the container in privileged mode. Default to Container.Privileged if left unset. More info: releases.k8s.io/HEAD/docs/design/security_context.md#security-context

  • runAsNonRoot (Property) -

    RunAsNonRoot indicates that the container should be run as a non-root user. If the RunAsUser field is not explicitly set then the kubelet may check the image for a specified user or perform defaulting to specify a user.

  • runAsUser (Property) -

    RunAsUser is the UID to run the entrypoint of the container process. The user id that runs the first process in the container. More info: releases.k8s.io/HEAD/docs/design/security_context.md#security-context

  • seLinuxOptions (Property) -

    SELinuxOptions are the labels to be applied to the container and volumes. Options that control the SELinux labels applied. More info: releases.k8s.io/HEAD/docs/design/security_context.md#security-context
